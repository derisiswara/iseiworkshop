Apa itu SEM?

  • Structural Equation Modeling = teknik statistik multivariat yang menggabungkan analisis faktor dan regresi
  • Memungkinkan pengujian hubungan simultan antara beberapa variabel
  • Dapat menguji variabel laten (tidak langsung teramati)
  • Digunakan luas di ilmu sosial, psikologi, manajemen, ekonomi

Terminologi Dasar

Istilah Penjelasan
Variabel Laten Konstruk yang tidak dapat diukur langsung (diberi simbol oval/lingkaran)
Variabel Manifes/Indikator Variabel yang dapat diukur langsung, misal item kuesioner (simbol kotak)
Variabel Eksogen Variabel independen (tidak diprediksi oleh variabel lain di model)
Variabel Endogen Variabel dependen (diprediksi oleh variabel lain di model)

Measurement Model vs Structural Model

  • Measurement Model: hubungan antara variabel laten dan indikator-indikatornya
    • “Seberapa baik indikator mengukur konstruk?”
  • Structural Model: hubungan antar variabel laten
    • “Bagaimana konstruk saling mempengaruhi?”
  • SEM menguji keduanya secara bersamaan

CB-SEM vs PLS-SEM

CB-SEM (Covariance-Based)

  • Berbasis kovarians
  • Memerlukan asumsi normalitas
  • Cocok untuk konfirmatori (teori sudah kuat)
  • Menggunakan fit indices
  • Paket R: lavaan

PLS-SEM (Partial Least Squares)

  • Berbasis varians
  • Tidak memerlukan normalitas
  • Cocok untuk eksploratori/prediktif
  • Tidak menggunakan fit indices tradisional
  • Paket R: seminr

Kapan Menggunakan yang Mana?

Kriteria CB-SEM PLS-SEM
Tujuan Konfirmasi teori Prediksi & eksplorasi
Ukuran sampel Besar (>200) Kecil-menengah (>50)
Distribusi Normal Fleksibel
Model Reflektif Reflektif & formatif
Kompleksitas Sederhana-menengah Bisa sangat kompleks

Apa itu CFA?

  • Confirmatory Factor Analysis = menguji apakah indikator mengukur konstruk dengan baik
  • Langkah pertama sebelum full SEM
  • Mengevaluasi validitas dan reliabilitas pengukuran
  • Berbeda dari EFA (Exploratory) yang mencari faktor; CFA mengkonfirmasi struktur yang sudah dihipotesiskan

Sintaks lavaan

Sintaks lavaan menggunakan notasi khusus:

Operator Makna Contoh
=~ “diukur oleh” (measurement) SQ =~ SQ1 + SQ2 + SQ3
~ “diregresikan terhadap” (structural) CS ~ SQ + KI
~~ kovarians/varians KL ~~ KI 
# CFA model
model <- '
  SQ =~ SQ1 + SQ2 + SQ3 + SQ4
  PV =~ PV1 + PV2 + PV3
'

Menspesifikasi Model CFA

cfa_model <- '
  # Measurement model
  SQ =~ SQ1 + SQ2 + SQ3 + SQ4
  PV =~ PV1 + PV2 + PV3
  TR =~ TR1 + TR2 + TR3 + TR4
  CS =~ CS1 + CS2 + CS3 + CS4
  CL =~ CL1 + CL2 + CL3
  RI =~ RI1 + RI2 + RI3
'

Mengestimasi Model CFA

cfa_fit <- cfa(cfa_model, data = ecom)

Fungsi cfa() dari paket lavaan:

  • Argumen pertama: spesifikasi model
  • data: dataset
  • Estimator default: ML (Maximum Likelihood)
  • Secara otomatis menetapkan loading pertama = 1 (untuk identifikasi)

Evaluasi Factor Loadings

  • Standardized loading (\(\lambda\)) \(\geq\) 0.50 (ideal \(\geq\) 0.70)
  • Loading < 0.50: indikator mungkin perlu dihapus
  • Semua loading harus signifikan (p < 0.05)
  • AVE (Average Variance Extracted) \(\geq\) 0.50 untuk convergent validity

Reliabilitas Konstruk

# Menghitung Composite Reliability (CR) secara manual
std_loadings <- parameterEstimates(cfa_fit, standardized = TRUE) |>
  filter(op == "=~") |>
  select(lhs, std.all)

std_loadings |>
  group_by(lhs) |>
  summarize(
    n_items = n(),
    sum_loading = sum(std.all),
    sum_loading_sq = sum(std.all^2),
    AVE = sum(std.all^2) / n(),
    CR = sum_loading^2 / (sum_loading^2 + (n() - sum(std.all^2)))
  ) |>
  kable(digits = 3)

Full Structural Model

Setelah CFA menunjukkan pengukuran yang baik, kita tambahkan jalur struktural (regresi antar laten).

sem_model <- '
  # Measurement model
  SQ =~ SQ1 + SQ2 + SQ3 + SQ4
  PV =~ PV1 + PV2 + PV3
  TR =~ TR1 + TR2 + TR3 + TR4
  CS =~ CS1 + CS2 + CS3 + CS4
  CL =~ CL1 + CL2 + CL3
  RI =~ RI1 + RI2 + RI3

  # Structural model (hipotesis)
  CS ~ SQ + PV + TR
  CL ~ CS + TR
  RI ~ CS + CL
'

Hipotesis Penelitian

No Hipotesis Path
H1 Service Quality berpengaruh positif terhadap Customer Satisfaction CS ~ SQ
H2 Perceived Value berpengaruh positif terhadap Customer Satisfaction CS ~ PV
H3 Trust berpengaruh positif terhadap Customer Satisfaction CS ~ TR
H4 Customer Satisfaction berpengaruh positif terhadap Customer Loyalty CL ~ CS
H5 Trust berpengaruh positif terhadap Customer Loyalty CL ~ TR
H6 Customer Satisfaction berpengaruh positif terhadap Repurchase Intention RI ~ CS
H7 Customer Loyalty berpengaruh positif terhadap Repurchase Intention RI ~ CL

Kepercayaan secara signifikan dipengaruhi oleh service quality (\(\beta\) = …, p < …) dan perceived value (\(\beta\) = …, p < …).

Bagian 3: PLS-SEM dengan seminr

Mengapa PLS-SEM?

  • Tidak memerlukan asumsi distribusi normal
  • Bisa menangani sampel kecil
  • Mendukung indikator formatif dan reflektif
  • Cocok untuk model yang kompleks dengan banyak variabel
  • Fokus pada prediksi (bukan konfirmasi teori)

Reflektif vs Formatif

Reflektif (efek)

  • Indikator adalah efek dari konstruk
  • Menghapus satu indikator tidak mengubah konstruk
  • Indikator saling berkorelasi tinggi
  • Contoh: item kuesioner kepuasan

Formatif (sebab)

  • Indikator membentuk konstruk
  • Menghapus satu indikator mengubah makna konstruk
  • Indikator tidak harus berkorelasi
  • Contoh: SES (pendidikan + pendapatan + pekerjaan)

Langkah PLS-SEM dengan seminr

  1. Spesifikasi measurement model (constructs())
  2. Spesifikasi structural model (relationships())
  3. Estimasi model (estimate_pls())
  4. Bootstrap (bootstrap_model())
  5. Evaluasi measurement model (outer model)
  6. Evaluasi structural model (inner model)

Spesifikasi Measurement Model

mm <- constructs(
  composite("IC", multi_items("IC", 1:4), weights = mode_B),
  composite("MO", multi_items("MO", 1:3), weights = mode_B),
  composite("EO", multi_items("EO", 1:3), weights = mode_B),
  composite("CA", multi_items("CA", 1:4), weights = mode_A),
  composite("FP", multi_items("FP", 1:4), weights = mode_A)
)
  • composite(): mendefinisikan konstruk
  • multi_items("prefix", 1:3): mengambil kolom prefix1, prefix2, prefix3
  • mode_A: reflektif (mode_B untuk formatif)

Spesifikasi Structural Model

sm <- relationships(
  paths(from = c("IC", "MO", "EO"), to = "CA"),
  paths(from = c("CA", "IC"), to = "FP")
)

Hipotesis:

  • H1: Innovation Culture \(\rightarrow\) Knowledge Management
  • H2: Market Orientation \(\rightarrow\) Knowledge Management
  • H3: Innovation Culture \(\rightarrow\) Kapasitas Inovasi
  • H4: Competitive Advantage \(\rightarrow\) Kapasitas Inovasi
  • H5: Competitive Advantage \(\rightarrow\) Kinerja Inovasi

Estimasi Model PLS

pls_model <- estimate_pls(
  data = inov,
  measurement_model = mm,
  structural_model = sm
)

Bootstrap

boot <- bootstrap_model(
  seminr_model = pls_model,
  nboot = 1000,
  seed = 42
)

Tip

Bootstrap menghasilkan standard error dan p-value untuk koefisien path. Gunakan minimal nboot = 1000 (ideal: 5000).

Apa yang Sudah Kita Pelajari

  • Konsep SEM: variabel laten, measurement vs structural model
  • CB-SEM dengan lavaan:
    • CFA: validitas dan reliabilitas konstruk
    • Full SEM: pengujian hipotesis antar konstruk
    • Fit indices: CFI, TLI, RMSEA, SRMR
  • PLS-SEM dengan seminr:
    • Measurement model: outer loadings, CR, AVE, HTMT
    • Structural model: path coefficients, bootstrap, \(R^2\)

Tips Akhir

  • Selalu mulai dengan CFA sebelum full SEM (CB-SEM)
  • Periksa asumsi sebelum menjalankan model
  • Laporkan semua fit indices yang relevan
  • Gunakan bootstrap untuk PLS-SEM (minimal 1000 resamples)
  • Jangan hanya mengandalkan p-value; perhatikan effect size (\(\beta\), \(f^2\), \(R^2\))
  • Teori harus mendahului analisis — jangan fishing for significance!
